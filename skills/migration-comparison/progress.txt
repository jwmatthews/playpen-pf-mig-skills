# Migration Evaluation System — Progress Log

## Codebase Patterns
- Always pass `--no-gumtree` when GumTree binary isn't available
- scoring-results.json uses `dir_a`/`dir_b` keys for resolved paths
- comparison-data.json `files.identical` items may be dicts with `path` key or plain strings
- ref_diff = inverted diff (golden content as + lines, AI content as - lines)
- cand_diff = as-is diff (golden content as - lines, AI content as + lines)
- Detectors check BOTH + and - lines for applicability via `_diff_has_pattern OR _diff_removes_pattern` — this can cause false applicability triggers when the AI uses a pattern that the golden truth doesn't
- Patterns perfectly applied by the AI (resulting in identical files between golden and AI) show as "not_applicable" since detectors only run on modified/removed files — this is by design and correct
- pydantic 2.11.7 is available system-wide (no venv needed)

## [2026-03-04] - Step 2: Review & Harden Deterministic Layer
- Ran evaluation against quipucords-ui PF5→PF6 migration:
  - Golden truth: quipucords/quipucords-ui PR #664 merge commit (4908c3b)
  - AI candidate: pranavgaikwad/quipucords-ui PR #1 head (a3ae2cf)
  - Result: B (80%) — File Coverage 100%, Pattern Score 73.8%, Noise Penalty 22%

- **Detector findings:**
  - 12 of 24 patterns fired (applicable), 12 not_applicable
  - Not_applicable patterns are genuinely not applicable — they were perfectly applied by both golden and AI, resulting in identical files (no diff = no detection needed)
  - 7 correct, 1 incorrect (select-rewrite), 4 missing (inner-ref-to-ref, is-action-cell, css-logical-properties, css-class-prefix partial)
  - `select-rewrite` INCORRECT confirmed: AI partially imported PF6 components (MenuToggle, SelectList) but still uses old API props (onToggle, isOpen, selections)
  - `inner-ref-to-ref` MISSING confirmed: golden truth changed innerRef→ref in 5 table-battery files, AI still uses innerRef
  - `is-action-cell` MISSING confirmed: golden truth changed some isActionCell→hasAction, AI didn't
  - Minor false positive: `button-icon-prop` marks AI as "correct" for using `icon=` on Button, but golden truth uses `PageToggleButton isHamburgerButton` instead — different approach, both valid PF6

- **Scoring assessment:**
  - B (80%) feels slightly generous given the select-rewrite failure, but reasonable
  - Composite score (C, 78%) with the adjusted weights (0.15 FC + 0.50 det + 0.10 noise + 0.25 LLM) better captures the quality when LLM results are added
  - Noise penalty correctly flags TODOs, unnecessary files, and incorrect migrations

- **Detector fix applied:**
  - `button-icon-prop`: changed applicability check from bidirectional diff matching to checking `ref_content` (golden truth file content) directly for `icon=` pattern. This fixes a false positive where the AI got credit for using `icon=` on viewLayout.tsx when the golden truth used a different approach (`PageToggleButton`). After fix, detector correctly finds 7 files (all genuine) instead of 2 (one false positive).

- **Known detector limitations:**
  - Some detectors still use bidirectional pattern matching (checks both + and - diff lines) for applicability, which can theoretically cause false triggers
  - Future improvement: apply the same content-based approach used in `button-icon-prop` fix to other detectors

- **Scoring assessment (after fix):**
  - Overall: B (80%) — same grade, pattern score stays 73.8%
  - Composite: C (78%) — appropriate for an AI migration with select-rewrite failure + several missed trivial patterns
  - Problem areas: 1 HIGH (select-rewrite incorrect), 3 LOW (css-logical-properties, inner-ref-to-ref, is-action-cell missing)
